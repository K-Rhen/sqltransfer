SQLtransfer

SQLtransfer is a small program written in Java, which can be used to transfer data from one database or directory-service to another database. Any database with an JDBC or ODBC driver can be used to retrieve or write data. Furthermore you can retrieve information from directory-services via LDAP. After the transfer it is possible to alter the information in the target database with the help of SQL-statements.

SQLtransfer can also be used for databases or drivers, which don't have an extended query-language or options. After transferring the data to a capable database, all wanted actions can easily be done in this database.

The program uses a SELECT-statement entered by the user to retrieve information from the source database. With the metadata of the resultset a table can be created in the target database and filled with the records form the source. All that is needed is a SELECT-statement and the name of the table in the target-database.

For the use of the program knowledge is required about making JDBC-connections to databases and knowledge about the SQL languages of the source and target database. If you are going to retrieve information from a directory-service, you also need to know about LDAP.

The program is configured with an XML-file. An explanation of the options can be found in the file sqltransfer_default.xml.

Working with SQLtransfer

What does SQLtransfer do

  • executes a given query in the source database
  • determines metadata of result
  • create a table with the columns and types in the metadata and with the given tablename in the target database
  • inserts all the records of the source result in the target table
  • executes the postqueries to alter the information in the target database

Logging of SQLtransfer

All actions of PrepareImportTables are logged in the log-file of the WHERE?. The queries executed and generated are also logged, so in case of problems one is able to determine the cause. Exceptions of the used databases are also shown here. Sometimes the exceptions can be unclear, because the used databases give us no clear information, which makes resolving problems not easy.

Installing a new JDBC-driver

New drivers for JDBC-connections can easily be added, so that support for new databases is possible. You must obtain the driver of the manufacturer. This driver (usually a jar-file) must be placed in the directory /lib of SQLtransfer. Edit BATCH

Edit shell-script

Edit batch-file

Find in wrapper.conf the lines starting with: 

wrapper.java.classpath
Add a new line here and increase the number of the last line with 1. Further add the filename and the directory at the end. For example the driver for PostGreSQL postgresql-8.0-311.jdbc3.jar in the directory lib: 
wrapper.java.classpath.4=lib/postgresql-8.0-311.jdbc3.jar

Edit batch-file

In the file settings.xml you can enter the new properties for the driver (driver-class, url, username, password). 

		<database>PostGreSQL</database>
		<jdbc>
			<driver>org.postgresql.Driver</driver>
			<url>jdbc:postgresql://HOSTNAME/DATABASENAME</url>
			<user>USERNAME</user>
			<password>PASSWORD</password>
		</jdbc>

Explanation of the files and directories

classes contains the class-files and source of SQLtransfer
lib contains the needed files for wrapper and JDBC-connections (jar-files)
readme.html this readme
sqltransfer.xml the file with the settings for SQLtransfer
wrapper_license.txt information about the license of the Wrapper
wrapper.conf in this file Wrapper is configured
wrapper.exe the program used to that executes the wrapper and runs SQLtransfer
wrapper.log the log-file of the wrapper, both Wrapper and SQLtransfer use this file for their logging
